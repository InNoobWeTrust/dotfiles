# Brainstorming

Structured ideation: frame → diverge → facilitate → converge → evaluate → action plan.

Phase detail and techniques: `references/phases.md`.

## Flow

1. **Frame** the problem (goal, constraints, success).
2. **Diverge** (volume techniques — SCAMPER, worst idea, etc.).
3. **Facilitate** (no early judgment; cluster themes).
4. **Converge** (rank, combine, kill weak ideas).
5. **Multi-perspective evaluate** (optional hats / stakeholders).
6. **Action plan** — top 3–5 with next experiments.

## Output (minimum)

Problem statement; raw idea dump; prioritized top 3–5 with rationale; next actions. Not a PRD — hand off to `requirements-driven-dev` when ready to specify.
